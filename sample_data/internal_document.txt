# 社内セキュリティ規定

## 1. 基本方針
1. CISO（情報セキュリティ責任者）を任命し、全社のセキュリティを統括
2. 各部門に情報セキュリティ管理者を配置し、月次で状況報告を実施
3. 情報資産を機密性に応じて3段階（高・中・低）に分類して管理
4. 四半期ごとにセキュリティ研修を実施し、受講状況を記録
5. 年2回のインシデント対応訓練を実施（6月・12月）

## 2. システム運用管理
1. 自動バックアップシステムを導入し、毎日23時に実行
2. バックアップデータの復旧テストを月1回実施
3. システムログは30日間保存し、週次で分析レポートを作成
4. セキュリティパッチは検証環境で1週間テスト後に本番適用
5. アクセス権限は人事異動の都度、および四半期ごとに棚卸を実施

## 3. インシデント対応
1. セキュリティインシデント発生時は情報システム部門に即時報告
2. 重大インシデントの場合、30分以内に経営層へ第一報を実施
3. インシデント対応チーム（CSIRT）が調査・対応を主導
4. 年次でインシデント対応手順の見直しと更新を実施

## 4. 禁止行為
1. 機密情報は社外へ持ち出し禁止（リモートアクセスは専用VPNを使用）
2. 業務システムの私的利用は禁止（ソーシャルメディアへのアクセスを含む）
3. 管理者の許可なくセキュリティ設定を変更することを禁止
4. 許可されていない外部記憶媒体の使用を禁止
5. 未承認ソフトウェアのインストールを禁止

## 5. 外部委託管理
1. 委託先の選定時にセキュリティ評価を実施
2. 委託先との契約にセキュリティ要件を明記
3. 年1回以上、委託先の遵守状況を監査
4. 重要システムの委託先に対して四半期ごとの状況報告を要求
